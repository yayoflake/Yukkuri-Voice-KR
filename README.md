# 윳쿠리 보이스 (한국어) 🗣️

한국어 텍스트를 **가타카나(음성기호열)** 로 변환해 **AquesTalk1** 윳쿠리 보이스로 읽어주는
**서버 없는 순수 브라우저 정적 앱**입니다. 백엔드도, 설치도, 회원가입도 없습니다.

### ▶ 바로 사용하기: **https://yayoflake.github.io/Yukkuri-Voice-KR/**

텍스트를 입력하고 ▶ 재생을 누르면 됩니다. PC·모바일 브라우저에서 모두 동작합니다.

> 모든 처리는 **브라우저 안에서** 일어납니다. 입력한 텍스트는 서버로 전송되지 않습니다.
> 음성 엔진은 페이지의 정적 파일에서 받아오므로 별도 외부 API 호출이 없습니다.

- 음성 엔진: [`aquestalk.js`](https://www.npmjs.com/package/aquestalk.js) — AquesTalk(Win32 DLL)를
  v86(x86 에뮬레이터, WebAssembly) 위에서 브라우저에서 구동.
- 일본어 G2P(한자 읽기) 불필요. AquesTalk는 **가타카나 + 액센트**를 그대로 받으므로,
  한국어를 가타카나로 근사 변환해 그대로 넘깁니다.

## 사용법

1. 위 주소에 접속합니다.
2. **읽을 한국어**를 입력하면 아래 **가나 칸**에 변환 결과가 실시간으로 채워집니다.
3. **음성(8종)** 과 **속도(50–300)** 를 고르고 **▶ 재생**.
4. 어색한 부분은 **가나 칸을 직접 수정**해 그대로 재생할 수 있습니다. (가타카나 ↔ 히라가나 토글 지원)

### 억양 · 장음 넣기

**한국어 입력칸**에서 바로 쓸 수 있고(권장), 가나 칸에 직접 넣어도 됩니다. 발음 규칙(연음·동화 등)은
그대로 유지되며, 기호는 앞 음절 뒤에 붙습니다.

- **억양(악센트핵)**: 음이 내려가는 자리에 아포스트로피 `'`. 예) `안녕'하세요` → `アンニョン'アセヨ`
- **장음(길게)**: 한국어 키보드에서 치기 쉬운 하이픈 `-`(또는 장음 기호 `ー`). 예) `아-` → `アー`, `네--` → `ネーー`
- **구간 속도·피치**: **가나 칸**에서 그 자리부터 다음 같은 태그까지 속도/음높이를 바꾼다.
  - 속도 `[숫자]`(50~300): 예) `[120]オハヨ[250]ゴザイマス` → 앞은 느리게·뒤는 빠르게
  - 피치 `{반음}`(-12~+12, 강세 올림): 예) `ガ{+4}ガ{0}ガ` → 가운데만 4반음 높게
  - 피치 한 단계 `>`(올림)·`<`(내림): 현재 피치에서 ±1반음(상대·누적). 예) `>ユックリ<ボイス`
    → ユックリ만 한 단계 높이고 다시 원위치. `>>`처럼 겹치면 그만큼.
  - 억양 초기화 `x`: `.`(`。`)처럼 끊고 억양을 리셋하되 **쉼 없이** 이어 붙인다. 예) `ボイスx입니다`
    → `x` 앞뒤를 따로 합성해(각자 새 억양) 무음 없이 크로스페이드로 연결. (길게 이어 읽어 억양이
    가라앉을 때 중간에서 음을 되살리는 용도. `.`는 쉼+리셋, `x`는 리셋만)

  AquesTalk1엔 구간 속도·피치 기능이 없다. 구간마다 따로 합성하면 호출마다 억양이 리셋돼
  문장이 끊기므로, **태그를 다 떼고 전체를 한 번에 합성**(연속 억양 확보)한 뒤 **시변(time-varying)
  위상 보코더로 전체를 한 번에 통과**시키며 표본별로 피치(신축+리샘플)·속도(시간신축)를 바꾼다.
  합성 위상이 끝까지 안 끊겨 **구간 경계의 클릭이 안 생기고**, 경계는 ~40ms 램프로 완만히 글라이드된다.
  외부 라이브러리는 안 쓴다(FFT·보코더 인라인 구현). 시작/끝엔 짧은 페이드로 재생·정지 클릭을 막는다.
  - **한계**: 음소 타이밍을 알 수 없어 구간의 시간 위치를 **모라(박자) 수 비례로 추정**한다 →
    강조 위치가 **±1모라** 밀릴 수 있다(구 단위로 쓰면 묻힘). 큰 변형엔 보코더 특유의 쇳소리.
  - 한국어 칸의 숫자는 읽는 수로 처리되므로 태그는 **가나 칸 전용**.

> 페이지를 처음 연 뒤 첫 재생 시, 음성 엔진(에뮬레이터)을 불러오느라 수십 초 걸릴 수 있습니다.
> 한 번 로드되면 이후엔 빠릅니다.

## 변환 규칙

**핵심: 받침은 음절이 아니라 코다(coda) 자음이다.** 모음을 붙여 `プ/ム/ク`로 만들지 않고
일본어의 발음(撥音) `ン` 또는 촉음 `ッ`으로 근사한다. 한글을 초성·중성·종성으로 분해한 뒤 적용:

1. **연음(連音)** — 받침이 있고 다음 초성이 `ㅇ`이면 받침을 다음 초성으로 넘김. (음악→으막)
   겹받침은 앞 자음만 남고 뒤 자음이 넘어감(닭이→달기). `ㅎ`은 모음 앞에서 탈락(좋아→조아).
   - **절음(絶音)** — **띄어쓰기(어절 경계)를 넘을 땐 받침을 7종 대표음으로 바꾼 뒤** 넘긴다.
     `옷 안에→오단에`(ㅅ→ㄷ), `샤인머스캣 알아→…캐달아`(`ケダラ`, ~~`ケサラ`~~), `닭 앞→다가파`.
     (한 단어 안에서는 원음 그대로: `옷이→오시`, `맛있어→마시써`)
2. **구개음화** — 넘어간 `ㄷ/ㅌ + ㅣ → ㅈ/ㅊ`. (같이→가치 → `カチ`)
3. **격음화** — `ㅎ ↔ ㄱ/ㄷ/ㅂ/ㅈ → ㅋ/ㅌ/ㅍ/ㅊ`. (좋다→조타 → `チョタ`)
   또한 **`ㅇ`받침 뒤 `ㅎ`은 약화·탈락**시킴. (융합→융압 → `ユンア`)
4. **유음화** — `ㄴㄹ`/`ㄹㄴ` 인접 → `ㄹㄹ`. (신라→실라 `シルラ`, 설날→설랄 `ソルラル`)
5. **비음화** — 파열음 받침 `+ ㄴ/ㅁ` → 비음. (**입니다→임니다 → `インニダ`**, 국물→궁물 `クンムル`)
6. **받침 → 가타카나** — 7종 중화 후 (받침에 가짜 모음 `루/부/프`를 붙이지 않는 게 원칙):
   - 비음 `ㄴ/ㅁ/ㅇ → ン`
   - 유음 `ㄹ → ン` (`ル`로 하면 "루" 모음이 끼어 어색). 단 `ㄹㄹ`(유음화) 연쇄만 `ル` 유지.
     예) 밥을→`パブン`, 신라 → `シルラ`
   - 파열음 `ㄱ/ㄷ/ㅂ → 촉음 ッ` (뒤에 음절이 올 때만). **어말에서는 떨어뜨림**
     (AquesTalk1이 어말 `ッ`을 거부하고, 가짜 모음도 안 붙이기 위해). 밥→`パ`, 음악→`ウマ`
7. **유성음화** — 모음·비음 사이의 `ㄱㄷㅂㅈ → 탁음 ガ/ダ/バ/ジャ`. (아버지→`アボジ`)
   어두/쉼 뒤에서는 무성음(보이스→`ポイス`).
8. **근사 매핑** — 없는 음은 가까운 가타카나로: `ㅓ→오`, `ㅡ→우`, `ㅐ/ㅔ→에`.
   합용모음(반모음 w)은 **단순화해서 단모음으로**: `과→カ`, `와→ア`, `위→イ`, `워→オ`.
   (AquesTalk1이 `クァ`류를 거부 — ERROR 105 — 하기 때문이기도 함)
   - **조사 `의` → `에`** — 관형격 조사 `의`는 표준발음상 [에]. 어절 끝의 `의`(ㅇ+ㅢ)만
     `エ`로 읽는다. (`주변의→チュビョネ`, `나의→ナエ`) 어두 `의`(의사→`イサ`)와 뒤에 조사가
     붙는 한자어 `의`(주의를→`チュイルン`)는 `イ` 유지. (단독 `회의`류는 `エ`가 되는 한계)
   - **외래음 `ティ/ディ/トゥ/ドゥ`** — 일본어 `タ行`은 i열이 `チ`·u열이 `ツ`라
     한국어 `티/디/투/두/트/드`가 `チ/ジ/ツ/ズ`로 뭉개진다. 원음을 살려 따로 매핑:
     `디오→ティオ`, `파티→パティ`, `라디오→ラディオ`, `두부→トゥブ`. (유성음화 규칙은
     그대로 적용되어 어두는 `ティ/トゥ`, 모음 사이는 `ディ/ドゥ`)
9. **구분자** — **띄어쓰기 → 어절 경계**: 받침이 다음 단어로 연음되면 **절음**(규칙 1)으로
   묶어 읽고(샤인머스캣 알아→`…ケダラ`, 책 읽기→`チェギッキ`), 그렇지 않으면 **악센트구 구분 `/`**
   (쉼 없이 억양만 분리)를 넣는다(전 가요→`チョン/ガヨ`, 밥 먹어→`パン/モゴ`). `ㅎ`약화는 어절을
   넘지 않음(안녕 하세요→`ハセヨ`). 유성음화·비음화는 경계를 넘어 유지.
   `, →、`, `. ! ? →。`, **줄바꿈 →。**. (쉼 `、。` 옆의 `/`는 잉여라 생략)

### 예외 사전 ([`dict.js`](dict.js))

규칙으로 못 잡는 **닫힌 예외**는 발음 사전으로 보정합니다. 입력의 **어절(공백·구두점으로 끊긴
한글 덩어리)이 사전 키와 정확히 일치할 때만** 등록된 발음으로 치환하고, 그 발음을 다시 규칙으로
가나로 바꿉니다.

- 예: 한자어 `의`는 [이]지만 규칙은 어절 끝 `의`를 조사로 보고 [에]로 읽음 → `회의→ヘイ`,
  `민주주의→ミンジュジュイ` 로 교정. (조사 붙은 `회의를`·복합어 `회의실`은 어절이 달라 규칙이 이미 맞힘)
- `WORD_OVERRIDES`에 `'단어': '발음(한글)'` 로 자유롭게 추가할 수 있습니다.
  (부분 일치는 안 하므로, 조사까지 포함한 형태가 필요하면 따로 등록)

> 목표는 "사람이 알아들을 정도". AquesTalk1은 일본어 엔진이라 `ㅓ/ㅡ`나 받침은 한계가 있고,
> 액센트는 기본값이라 억양은 어색할 수 있습니다.
> 어색한 단어는 화면의 **가나 칸을 직접 고쳐서** 재생할 수 있습니다.
